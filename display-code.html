Display Code with Syntax Highlighting

Problem

The Canvas page editor provides <pre> tags, but content within those tags is not rendered with syntax highlighting. However, gist.github provides a javascript a script which can be embedded in a page. This is not a complete solution, however, as Canvas does not allow using <script> in the Canvas HTML editor.

Solution

The workaround is to:

  1. Inject the appropriate <script> tags in an HTML file
  2. Save that HTML file in the Canvas file section
  3. Load that HTML file into the Canvas page with an iFrame

The following is an example of the HTML file that would be loaded into the Canvas files section (we've loaded this sample via the iFrame function). You may need to experiment with the height to display the entire frame without scroll bars (developer tools can assist to identify the height of the iFrame). Note the automagic syntax highlighting. The js script is the script provided by gist.github and returns the formatted code element for the page. Changes made to the gist are automatically reflected here.
