Render Code stored in repository

Problem

Code reuse principles warrant storing code snippets apart from the way they are used or are presented. Github is a useful place to maintain code smippet repositories, but there is not a straightforward way to reuse those snippets in Canvas.

Solution

Github does not allow extracting the raw code from its server to be able to reuse elsewhere, within an iFrame or otherwise. The workaround involves working with a third party API which extracts the code using the github API and then wraps that code with the approprate Content-Type headers.

The workaround is to:

  1. Find the raw file text url that you want to include. This can be a file on the main branch of a github repositoty, a feature branch, a tagged version, or a github gist.
  2. Enter that url into raw.githack.com
  3. Select the appropriate url (production or development)
  4. Use that url within an iFrame
  5. If you used a development url, be sure to update the url to a production url after you're satisfied with the source code. If necessary, you can purge cached urls if you make a change to source code referenced by a production url
